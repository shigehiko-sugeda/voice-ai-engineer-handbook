# Contributing

Voice AI Engineer Handbookへの改善提案と教材追加を歓迎します。このリポジトリは、車載音声認識を学ぶ人が、設計・ログ・品質・安全性を実務に近い粒度で確認できることを目指しています。

## 基本方針

- 日本語で簡潔に書き、専門用語は初出時に説明する。
- 教材本文に沿い、車載音声認識の文脈を優先する。
- 不確かな内容を断定せず、仮定または要確認事項と明記する。
- 社外秘情報、個人情報、特定企業の非公開仕様を含めない。
- 認証の詳細は公開情報と参照可能な正式資料の範囲に留める。
- 正常系だけでなく、Timeout、キャンセル、競合、Fallback、Recoveryを扱う。

## フォルダの役割

- `docs/`: 技術の責務、詳細、車載例、設計・レビュー観点を説明する。
- `diagrams/`: 処理順序、状態遷移、所有権、Process境界を図示する。
- `cheatsheets/`: 設計や不具合調査で確認する項目を短くまとめる。
- `quizzes/`: 章ごとの理解度を確認する。
- `exercises/`: ケース、設計、ログ解析を通して判断を練習する。
- `prompts/`: AI家庭教師へ渡す再利用可能な指示を管理する。
- `templates/`: 新しい教材を既存形式に合わせて作る。

同じ説明を複数フォルダへ複製しないでください。処理の流れは`diagrams/`、判断項目は`cheatsheets/`、詳細な説明は`docs/`を正本とします。

## 教材を追加・変更する手順

1. [教材作成テンプレート](templates/README.md)から目的に合う形式を選ぶ。
2. 関連する既存章、図、チートシート、クイズ、演習を確認する。
3. 入力、判断・処理、出力と、前後Componentの責務境界を書く。
4. 車載固有の制約と、判断に必要なログを追加する。
5. 関連ファイルを相互リンクし、リンク先が存在することを確認する。

新しい章を追加する場合は、原則として次も更新してください。

- `README.md`の章一覧と学習順序
- `docs/00_learning-roadmap.md`
- 対応する`quizzes/`の章別クイズ
- 必要に応じて`diagrams/`、`cheatsheets/`、`exercises/`
- `docs/glossary.md`の関連用語

## 章の構成

章は[章テンプレート](templates/chapter-template.md)に従い、少なくとも次を含めます。

- この章で学ぶこと
- なぜ必要なのか
- 全体像と基本用語
- 詳細解説と車載での利用例
- 設計時の注意点
- レビュー観点
- よくある不具合
- 確認問題、まとめ、関連クイズ

## 設計レビュー6観点

設計レビューでは、指摘と確認質問を次の観点に分けます。

- **機能**: 正常系、状態遷移、キャンセル、Retry。
- **品質**: 精度、Latency、CPU、Memory、測定条件。
- **ログ**: 関連ID、State、Event、時刻、失敗理由、版情報。
- **認証**: Voice Button、Wake Word、Mic Routing、Audio Focus、証跡。
- **安全性**: 走行中制限、誤操作、値域、権限、低Confidence時の確認。
- **運用**: Timeout、監視、Fallback、Recovery、切り戻し。

## 参考資料

- 技術的な説明には、可能な範囲で公式仕様、標準、一次資料を示す。
- ブログや二次資料だけを根拠に重要な仕様を断定しない。
- 引用は必要最小限にし、本文は自分の言葉で説明する。
- 外部Assistantや認証に関する内容は、公開ページと非公開資料を混同しない。

## Pull Request前の確認

- [ ] `git diff --check`が成功する。
- [ ] 追加・変更したMarkdownの相対リンクが切れていない。
- [ ] Mermaid図がGitHub上で表示できる構文になっている。
- [ ] 章と対応クイズのファイル名・リンクが一致している。
- [ ] プレースホルダー、TODO、未記入欄が残っていない。
- [ ] 社外秘情報、個人情報、非公開認証仕様を含んでいない。
- [ ] 機能、品質、ログ、認証、安全性、運用の観点を確認した。

変更は目的ごとに分け、関係のないファイルを同じCommitへ含めないでください。
