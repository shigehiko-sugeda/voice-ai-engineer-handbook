# Promptガイド

車載Voice AIの学習、出題、設計レビュー、面接練習に使うPrompt集です。各ファイルの「入力」部分を埋め、Prompt本文と一緒にAIへ渡してください。

## 一覧

| Prompt | 用途 | 主な入力 |
|---|---|---|
| [初心者向け説明](explain_beginner.md) | 章を段階的に理解する | 対象章、経験、疑問点 |
| [クイズ生成](generate_quiz.md) | 理解度を確認する | 対象章、問題数、難易度 |
| [設計演習生成](generate_design_exercise.md) | ケース形式で設計を練習する | テーマ、難易度、重点観点 |
| [設計レビュー](generate_review.md) | 設計案のリスクを抽出する | 設計内容、前提、重点観点 |
| [面接練習](interview.md) | 段階的な模擬面接を行う | 経験、職種、時間、重点分野 |

## 共通方針

- リポジトリ内で使う場合は、`AGENTS.md`と対象の`docs/`を参照する。
- 教材本文にある事実と、追加の一般知識や仮定を区別する。
- 最初から答えを提示せず、確認質問またはヒントから始める。
- 車載では、走行中制限、音声競合、Timeout、Fallback、ログ、復旧を考慮する。
- 認証の詳細は公開情報と参照可能な正式資料の範囲に留め、非公開仕様を推測しない。
- 設計レビューは、機能、品質、ログ、認証、安全性、運用に分ける。

## 使い方

1. 入力欄の`<...>`を置き換える。
2. 不明な欄は「未指定」と書き、AIに必要な確認質問をさせる。
3. 最初の回答ではヒントまたは質問だけを受け取る。
4. 自分の回答後に、採点、解説、改善案を依頼する。

設計演習の完成例は[`exercises/`](../exercises/README.md)を参照してください。
