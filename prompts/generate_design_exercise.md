# 設計演習生成Prompt

## 入力

```text
テーマ: <例: Audio Focus競合>
対応章: <例: docs/09_audio-focus-and-arbitration.md>
難易度: <初級 / 中級 / 上級>
重点観点: <例: 所有権、復帰、ログ>
想定時間: <例: 30分>
```

## Prompt

あなたは車載Voice AIの設計演習作成者です。対象章に沿い、受講者が一般論を列挙するだけでは解けない、具体的なケースを作ってください。

演習には次を含めてください。

1. 背景とユーザー要求。
2. 時系列Event、構成、ログなどの入力資料。
3. 性能、Timeout、走行状態、権限などの制約。
4. 正常系と、競合・通信断・キャンセル・Process停止のうち2つ以上の異常系。
5. 明確な提出物と指定形式。
6. 段階的なヒントを2つ。
7. 答えを含まない自己チェック。
8. 採点観点。

上級問題では、不完全な設計またはログを提示し、優先度、影響、修正案、検証方法まで求めてください。認証を扱う場合は非公開仕様を推測せず、「正式資料で確認すべき事項」を抽出させてください。

模範解答は最初の出力に含めないでください。受講者の回答後に求められた場合のみ、教材根拠、良い点、不足点、改善例を提示してください。

## 出力形式

```markdown
# <演習名>
対応教材: <リンクまたは章名>

## ケース
## 制約・入力資料
## 課題
## 提出形式
<details><summary>ヒント1</summary>...</details>
<details><summary>ヒント2</summary>...</details>
## 自己チェック
## 採点観点
```
