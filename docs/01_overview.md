## 詳細解説

車載音声認識は、ユーザーの発話を車両機能の実行や情報提示につなげるパイプラインです。

```text
音声入力
  ↓
Wake Word
  ↓
ASR
  ↓
NLU
  ↓
Dialog Manager
  ↓
Skill / Domain
  ↓
Vehicle API
  ↓
TTS
```

ASRは「音声を文字にする」役割です。NLUは「文字列から意味を理解する」役割です。Dialog Managerは「会話状態を管理し、次に何を聞くか・実行するかを決める」役割です。Skill / DomainとVehicle APIは「車両機能を実行する」接点です。TTSは結果や確認を音声で返します。
