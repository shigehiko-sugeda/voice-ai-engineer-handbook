# 音声・オーディオ基礎

## この章で学ぶこと

- サンプリング、ノイズ、マイク、エコーの基礎
- 車室内音響がASRへ与える影響
- 音声入力の問題をログと録音で切り分ける方法

## なぜ必要なのか

ASRへ渡る前の音が欠けたり歪んだりしていれば、モデルだけを調整しても改善しません。車室内では走行音、空調、音楽、反響、座席位置が同時に影響するため、音声入力と認識処理を分けて評価する必要があります。

## 全体像

```text
話者 → マイク → A/D変換 → Beamforming・AEC・Noise Reduction → VAD → ASR
                         ↑ スピーカー再生音
```

## 基本用語

- **Sampling Rate**: 1秒間に音を測定する回数。入出力間で一致させる。
- **SNR**: 目的音声と雑音の強さの比。
- **VAD**: 音声区間を検出する処理。
- **AEC**: スピーカー再生音がマイクへ回り込むエコーを抑える処理。
- **Beamforming**: 複数マイクから特定方向の音を強調する処理。

## 詳細解説

音声品質は「聞こえるか」だけでなく、欠落、クリッピング、遅延、チャネル、フォーマットで確認します。前処理を強くしすぎると雑音と一緒に子音が失われ、弱すぎるとASRへ雑音が残ります。AECでは、実際に再生した信号を参照信号として正しい時刻で渡す必要があります。

評価時は加工後の音だけでなく、可能な範囲でマイク入力、AEC参照、前処理後の音を対応づけます。音声データの保存は同意、匿名化、保存期間、アクセス権を定めます。

## 車載での利用例

高速道路で運転席から「次のサービスエリア」と発話した場合、ロードノイズ、助手席の会話、ナビ音声が重なります。運転席方向を強調し、ナビ再生音をAECで抑え、VADが発話の開始と終了をASRへ通知します。

## 設計時の注意点

- マイク数、配置、対象座席、入力チャネルを仕様化する。
- サンプリングレート、ビット深度、フレーム長、時刻基準を統一する。
- 音楽、TTS、ナビ、通話ごとにAEC参照の有無を確認する。
- 停車・市街地・高速・雨・窓開け・空調強度を評価条件に含める。

## レビュー観点

- **機能**: 対象話者を拾い、再生音を発話として扱わないか。
- **品質**: SNR、欠落、クリッピング、音声区間検出を測定できるか。
- **ログ**: デバイス、チャネル、フォーマット、フレーム時刻、VAD状態を追えるか。
- **運用**: マイク故障や無音を検知し、安全に機能を停止できるか。

## よくある不具合

- 入出力のSampling Rate不一致で再生速度や認識率が変わる。
- AEC参照の遅延ずれでTTSをユーザー発話と誤認する。
- 前処理が強すぎて発話の先頭や子音が消える。
- マイク故障によるゼロ値を「無発話」として処理し続ける。

## 確認問題

1. TTS再生中だけ誤認識が増える場合、どの信号とログを比較しますか。
2. 高速走行時の評価条件を3つ挙げてください。
3. 音声を保存する場合に必要な運用上の配慮は何ですか。

## まとめ

ASR品質の土台は正しい音声入力です。車室内条件を分けて録音・前処理・VADを評価し、フォーマットと時刻をログで追跡できるようにします。
